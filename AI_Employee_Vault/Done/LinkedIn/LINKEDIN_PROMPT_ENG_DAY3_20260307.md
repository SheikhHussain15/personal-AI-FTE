---

### 📌 The Two Types of Prompts

**1. SYSTEM INSTRUCTIONS** (The Setup)
**2. USER PROMPTS** (The Request)

Understanding the difference is a game-changer.

---

### 🎭 What Are System Instructions?

**System instructions** are the "behind-the-scenes" directions you give AI BEFORE your actual prompt.

Think of it as:
- **System** = Setting the stage, rules, and character
- **User** = The actual scene/dialogue

**Analogy:**
- System = Telling an actor "You're a wise old mentor"
- User = The lines they say in character

---

### ❌ Common Mistake

Most people ONLY use user prompts:

```
USER: "Write a Python function to sort a list"
```

This works... but it's leaving performance on the table.

---

### ✅ The Pro Approach

**SYSTEM + USER = Exceptional Results**

```
SYSTEM: "You are a senior Python developer who writes clean, efficient code. You always include docstrings and type hints. You explain your reasoning."

USER: "Write a Python function to sort a list"
```

**Result:** Better code, better explanations, better everything.

---

### 🔍 Side-by-Side Comparison

**User Prompt Only:**
```
"Explain quantum computing"
```
*Result: Generic Wikipedia-style explanation*

**System + User:**
```
SYSTEM: "You are a physics professor teaching high school students. Use simple analogies. Avoid jargon. Make it fun and engaging."

USER: "Explain quantum computing"
```
*Result: Clear, age-appropriate explanation with relatable analogies*

---

### 💡 Where to Use System Instructions

**1. Chatbots & AI Assistants**
Set the personality and expertise level once, then ask many questions.

**2. Content Creation Workflows**
Define your brand voice as system, then generate multiple pieces.

**3. Code Generation**
Set coding standards as system, then request specific functions.

**4. Customer Service**
Define tone and policies as system, then handle individual cases.

---

### 📝 System Instruction Templates

**For Content Creation:**
```
"You are a professional [ROLE] with [X] years of experience. Your writing style is [STYLE]. You always [BEST PRACTICE]. Your audience is [AUDIENCE]."
```

**For Code:**
```
"You are a senior [LANGUAGE] developer. You write [QUALITIES] code. You always include [ELEMENTS]. You follow [STANDARDS]."
```

**For Analysis:**
```
"You are an expert [FIELD] analyst. You think critically and question assumptions. You provide evidence-based conclusions. You acknowledge uncertainty."
```

---

### 🎯 Pro Tips

**1. Be Specific About Expertise:**
- ❌ "You are helpful"
- ✅ "You are a certified financial planner with 15 years of experience"

**2. Define the Thinking Process:**
- ✅ "Think step-by-step before answering"
- ✅ "Consider multiple perspectives before concluding"

**3. Set Quality Standards:**
- ✅ "Only provide information you're confident is accurate"
- ✅ "If unsure, say 'I don't know' rather than guessing"

**4. Specify the Relationship:**
- ✅ "Act as my personal coach"
- ✅ "You are my research assistant"

---

### 🔥 Real Example: Email Writing

**SYSTEM:**
```
"You are a professional business communicator. Your emails are concise, warm, and action-oriented. You always include a clear call-to-action. You write at a 10th-grade reading level."
```

**USER:**
```
"Write an email to a client explaining that we need 2 more weeks for their project due to unexpected technical challenges."
```

**Result:** Professional, empathetic, clear email that maintains trust.
