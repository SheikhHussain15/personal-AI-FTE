# AI Employee Vault Operations

Bronze Tier agent skill for managing the Obsidian vault and processing tasks.

## Quick Start

```bash
# Navigate to vault
cd AI_Employee_Vault

# Run orchestrator to check status
python Scripts/orchestrator.py --vault ./AI_Employee_Vault

# Start filesystem watcher
python Scripts/filesystem_watcher.py --vault ./AI_Employee_Vault
```

## Core Capabilities

### 1. Process Needs_Action Items

When files appear in `/Needs_Action`:

1. **Read** the file and any associated `.meta.md` file
2. **Analyze** what action is needed
3. **Check** Company_Handbook.md for rules
4. **Create** a plan in `/Plans/` if multi-step
5. **Execute** actions within Bronze tier scope:
   - Organize and categorize
   - Extract information
   - Create summaries
   - Flag for approval if needed
6. **Move** completed items to `/Done`

### 2. Update Dashboard

Keep Dashboard.md current:

```bash
# Update stats
python Scripts/orchestrator.py --vault ./AI_Employee_Vault
```

Dashboard sections to maintain:
- Quick Stats (pending, completed, approvals)
- Recent Activity
- Financial Snapshot
- Key Metrics

### 3. File Organization

Standard folder structure:

```
AI_Employee_Vault/
├── Inbox/              # Drop zone for new files
├── Needs_Action/       # Items awaiting processing
├── Done/               # Completed items
├── Plans/              # Multi-step plans
├── Pending_Approval/   # Awaiting human decision
├── Approved/           # Approved for action
├── Briefings/          # CEO briefings
├── Accounting/         # Financial records
├── Logs/               # System logs
└── Scripts/            # Python scripts
```

### 4. Approval Workflow

For sensitive actions:

1. Create file in `/Pending_Approval/`:
```markdown
---
type: approval_request
action: [action_type]
created: [timestamp]
status: pending
---

# Approval Required

## Details
- Action: [description]
- Reason: [why approval needed]

## To Approve
Move this file to `/Approved` folder.

## To Reject
Move this file to `/Rejected` folder.
```

2. Wait for human to move file
3. If moved to `/Approved`, execute action
4. If moved to `/Rejected`, log and discard

|--------|--------------|------------------|

| Files | Create, read | Delete, move outside vault |

## Error Handling

### Common Issues

| Issue | Solution |
|-------|----------|
| File locked | Wait and retry (max 3 times) |
| Missing metadata | Create basic metadata from file info |
| Unknown file type | Flag for human review |
| Dashboard update fails | Log error, continue processing |

### Retry Logic

```python
def with_retry(func, max_attempts=3):
    for attempt in range(max_attempts):
        try:
            return func()
        except Exception as e:
            if attempt == max_attempts - 1:
                raise
            time.sleep(2 ** attempt)  # Exponential backoff
```
